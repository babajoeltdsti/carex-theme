# Carex Release Checklist

## Before Publishing

1. Review all theme variants in VS Code theme selector.
2. Validate dark, light, OLED, neon, and accessibility variants visually.
3. Check semantic highlighting in TypeScript, JavaScript, Python, Markdown, JSON, CSS, and TSX.
4. Check markdown headings, quotes, inline code, and frontmatter.
5. Check JSX tags, JSX attributes, JSON keys, CSS selectors, Python decorators, regex, and escapes.
6. Validate notebooks, sticky scroll, testing UI, merge editor, git decorations, and inline chat surfaces.
7. Confirm Carex File Icons appear for source folders, config folders, docs, tests, and major file types.
8. Confirm Carex Product Icons load without missing glyphs.
9. Run the package command and verify the produced VSIX opens correctly.
10. Update README and CHANGELOG if the release includes user-visible changes.
